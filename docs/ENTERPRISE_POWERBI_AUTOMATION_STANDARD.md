# 企业级 Power BI 自动化能力标准

## 1. 目的与权威性

本文是本仓库判断 Power BI 自动化能力、企业交付状态和对外声明的唯一标准。项目说明、作品集、Upwork Proposal 和进度文档不得使用高于本文证据等级的措辞。

当前建设目标是 **Desktop 可交付**：能够从获准使用的数据出发，形成可复现、可验证、可由客户在 Power BI Desktop 打开的工程交付包。Power BI Service、Fabric、网关、共享和真实多用户权限仍是独立质量门。

## 2. 成熟度等级

| 等级 | 名称 | 最低证据 | 允许声明 | 不允许声明 |
|---|---|---|---|---|
| L0 | 实验样板 | 合成或公开数据、局部模型或页面、实验记录 | 原型、实验、技术验证 | 客户可交付、生产级 |
| L1 | Desktop 可交付 | 授权与范围、数据合同、可刷新模型、DAX核对、RLS测试、报表QA、性能证据、PBIP/PBIX及交接文档 | 企业Desktop报表交付、AI辅助自动化 | Service已部署、生产运维、任意数据一键生成 |
| L2 | Service 可交付 | L1全部证据，加工作区发布、网关或云连接、计划刷新、真实共享/RLS、失败通知、部署与回滚记录 | Power BI Service部门级交付 | Fabric平台治理、跨租户通用能力 |
| L3 | 企业平台/Fabric | L2全部证据，加开发/测试/生产隔离、CI/CD、容量、审计、信息保护、监控和支持责任 | 企业平台或Fabric实施能力 | 未经实测的跨环境或跨组织承诺 |

等级按项目逐一判定，不因另一个项目通过而自动继承。当前仓库的企业销售自动化MVP目标为 **L1**。

## 3. 企业质量门

| 质量门 | 必须回答的问题 | 最低证据 | 失败处理 |
|---|---|---|---|
| 需求 | 支持谁做什么决定，交付什么 | 范围、验收标准、排除项 | 暂停开发 |
| 授权 | 数据能否处理、交付和公开 | 来源、许可、脱敏与公开状态 | 禁止导入或公开 |
| 数据合同 | 粒度、主键、字段和时间范围是否明确 | 机器可读合同与说明 | 阻断建模 |
| 数据质量 | 缺失、重复、异常、取消和边界是否处理 | 自动检查与异常清单 | 阻断或经业务负责人书面接受 |
| 指标 | 公式、筛选、分母、时间窗口是否明确 | 指标字典与独立基线 | 阻断报表 |
| 模型 | 星型结构、关系和筛选传播是否正确 | 模型规范、TMDL/PBIP和DAX查询 | 阻断报表 |
| RLS | 权限规则是否真实、可测试且不越权 | 角色定义、测试用户与查询证据 | 阻断交付 |
| 报表 | 页面是否支持已批准的决策链 | 设计规格、PBIR校验和人工截图审查 | 阻断交付 |
| 性能 | 刷新和交互是否达到项目阈值 | 刷新计时、Performance Analyzer或等价记录 | 优化或记录不接受 |
| 交付 | 客户能否打开、维护和理解 | PBIP/PBIX、源文件、使用与部署说明 | 阻断签收 |
| 作品集 | 是否真实、授权且不误导 | 分类、来源、限制和个人贡献 | 禁止公开 |

质量门状态只能使用 **pending**、**passed**、**failed** 或 **blocked**。没有新鲜验证证据时不得写 **passed**。

## 4. 自动化与人工责任

### 可以自动化

- schema检查、数据合同和质量测试；
- SQL、Power Query、TMDL、DAX和PBIR等文件化资产；
- 指标基线、DAX查询、RLS测试和PBIR结构校验；
- 证据文件、数据字典、交付清单和限制说明；
- 在已验证模板与已批准字段映射范围内实例化报表。

### 必须人工负责

- 客户需求、业务决策和范围确认；
- AI字段映射、指标口径和例外处理审批；
- 凭据、权限、客户环境和敏感数据授权；
- 视觉、交互、业务洞察和最终数字验收；
- PBIX最终保存、Service/Fabric发布和客户签收。

“一键”表示一条受质量门约束的编排命令，不表示跳过确认、错误阻断或人工验收。

## 5. 证据与措辞

所有重要结论使用以下标签：

- **事实**：来自当前数据、代码、工具输出、正式文档或验证结果。
- **推断**：由事实支持但尚未获得业务负责人确认。
- **未知**：当前缺少证据。
- **建议**：基于目标和风险提出的下一步。

### 禁止声明

- 把单一销售模板描述成适用于任意行业、任意schema的一键工具；
- 在没有成本字段时估算或虚构利润、毛利和毛利率；
- 把模拟用户—国家映射描述成真实客户权限；
- 把部署手册描述成已完成Service、网关或多用户共享验证；
- 把公开数据练习描述成真实付费客户项目；
- 使用 **production ready**、**fully automated** 或“企业生产已验证”等超出证据的措辞。

### 推荐声明

> Public-data portfolio project demonstrating a governed, template-based Power BI Desktop delivery workflow. Service deployment, gateway configuration, and real client acceptance are outside the validated scope.

## 6. 失败、覆盖与证据留存

- 数据合同、字段映射、指标、RLS、PBIR或性能质量门失败时，流程必须返回非零结果或明确的 **blocked** 状态。
- 失败流程不得覆盖最后一个健康PBIP/PBIR基线。
- 每个质量门保存机器可读JSON和人类可读Markdown；视觉质量门另存经过检查的截图。
- 修复后重新运行完整相关质量门，不使用旧证据代替当前结果。
- 发布、发送、权限修改、commit 和 push 继续要求单独授权。
