# Light Executive 报表 UI 设计系统

## 设计目标

面向管理者的浅色企业界面：先看关键经营数字，再看趋势与驱动因素，最后定位国家、客户或产品。它服务于公开数据练习和作品集演示，不模仿或复制任何企业品牌。

## 视觉令牌

| 类别 | 规则 |
|---|---|
| 画布 | `#F7F9FC`，避免纯白大面积眩光。 |
| 主色 | `#164E8C`，用于页面标题、关键趋势和活动状态。 |
| 强调色 | `#009E8A`，用于正向强调，不单独表达“好”。 |
| 警示色 | `#C2410C`，用于取消/风险信息，并配合文字。 |
| 正文 | `#1F2937`；次级文字 `#667085`。 |
| 字体 | Segoe UI；页面标题 28px，区块标题 15px，KPI 标签 12px。 |
| 圆角与留白 | 卡片和切片器 8px 圆角；页面边距 32px；组件间距至少 16px。 |

令牌的机器可读来源为 [`../report/design-tokens.json`](../report/design-tokens.json)。

## 组件规则

- **顶部区**：页面标题在左，统一下拉筛选器在右；不新增网页式侧边栏。
- **KPI 卡片**：单个主指标、清楚的英文标签、无重边框；比较指标缺少可比期间时显示 `--`，不显示 0。
- **图表**：主趋势使用主色；排名条形图使用较浅的同色系；取消相关指标使用警示色；图表标题说明业务问题而非字段名。
- **切片器**：保持原生 Dropdown；显示“所有”与清除状态，最小高度 76px。
- **RLS 说明**：保留“Simulated RLS”声明，避免把公开练习误解为真实客户权限。
- **空状态**：不以残留趋势或 0 冒充结果；后续需补充“当前筛选范围无数据”的实测状态证据。

## 页面层级

1. `Executive Overview`：总体 KPI、月度趋势、国家排名。
2. `Product & Trend Analysis`：产品与时间驱动因素；MoM/YoY 只在单一可比月份显示。
3. `Customer & Country Analysis`：客户和国家贡献，并展示模拟 RLS 限制说明。

## 可访问性边界

- 正文与背景目标对比度至少 4.5:1。
- 颜色不得成为唯一状态信号；标题、数值和说明文字同时表达含义。
- Tab 顺序、屏幕阅读器、键盘操作和高对比度模式必须在 Desktop 中单独验证；本设计文档不构成通过证据。
