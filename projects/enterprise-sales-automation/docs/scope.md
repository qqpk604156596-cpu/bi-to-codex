# 项目范围

## 业务目标

为模拟的销售运营负责人提供一条三页管理决策链，回答整体销售表现、产品与趋势驱动、客户与国家表现，并验证模板化Power BI Desktop开发能否在明确质量门下完成。

## 用户与决策责任

- 报表用户：销售经理、运营经理、国家/区域负责人。
- 项目操作者：项目所有者，通过VS Code和Codex运行。
- 决策负责人：项目所有者。
- 首要外部读者：Upwork客户。

## 范围内

- UCI Online Retail II公开数据的来源、许可和数据质量审计；
- 已加载MySQL后的schema检查与AI字段映射建议；
- 人工确认字段映射、粒度、指标和模块降级；
- MySQL Import星型模型、10项销售指标和动态国家RLS；
- 三页PBIR模板实例化、DAX与DuckDB/SQL核对；
- PBIR校验、Desktop打开、刷新与交互性能证据；
- PBIP、人工确认后的PBIX、SQL、配置、证据和说明文档；
- 英文对外交付说明与中文内部维护说明。

## 范围外

- 下载UCI数据、创建MySQL表和生成报表不属于当前文档阶段；
- 利润、毛利、预算、目标和预测；
- 六页套件和三套不同schema复用；
- DirectQuery、SQL Server和PostgreSQL；
- Power BI Service、Fabric、网关、真实共享和真实用户RLS；
- 非技术客户GUI和任意数据自动理解；
- 把本项目描述为付费客户经历。

## 计划交付物

- 可复现的MySQL到Power BI Desktop生成流程；
- 三页英文销售运营报表；
- 数据合同、指标字典、模型与RLS说明；
- 自动验证和人工QA证据；
- 完整工程交付包与Upwork案例说明。

## 依赖与未知

- 原始数据必须按CC BY 4.0要求署名。
- MySQL schema、批准字段映射、数据合同和指标基线已验证。
- Power BI Desktop模型、刷新、代表性交互性能和最终PBIX已验证并留证。
- 字段映射确认后30分钟内完成的端到端效率目标没有计时证据，保持未验证且不作为本轮L1声明。
- 模拟RLS能证明机制，不能证明真实客户权限治理。
- Service与Fabric能力需要后续许可和测试环境。
